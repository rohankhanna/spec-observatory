# Research Scope Spec

## Scope

The observatory tracks the state of the art for:

- spec-driven development
- outcome-driven development
- coding-agent execution and agent harnesses
- orchestration and interoperability
- software portfolio and project steering

## Research Rule

The daily loop should prefer primary or official sources and should only update the report when the evidence changes the practical picture in a material way.

## Material Changes

Material changes include:

- a new major tool or category
- a deprecation or major repositioning
- a stronger or weaker practical conclusion
- a correction to an inaccurate claim
- a better abstraction boundary for understanding the stack

## Expected Output

The observatory report should answer:

- what categories matter right now
- which tools meaningfully define those categories
- what vocabulary is established versus invented
- where the practical abstraction boundaries sit between end-user agents, vendor harnesses, frameworks, protocols, and steering surfaces
- what the most credible shape of the ecosystem looks like
